The thesis

Today's agents reach the model only through a stateless text API — discarding the KV-cache, the logits, the draft proposals, the verifier scores. The gap that matters for local agents is architectural, not parameters. Crucible runs the agent loop below that wall and uses those control surfaces directly.

What ships today — Product 1: Crucible Core

Three token-savers. Real measurements.

0%
malformed tool calls

Grammar-scoped emission

The model emits actions under a GBNF grammar, so a tool call is structurally incapable of being malformed. Unconstrained decoding produced 100% malformed calls on the same suite; Crucible produces 0%.

4.28×
fewer tokens, same answers

Chain-of-Draft reasoning

A reasoning dial trades verbose chain-of-thought for terse drafts on easy steps: 152.5 → 35.7 mean completion tokens at matched 100% success (95% CI 3.4–5.5×).

4.94×
less prefill compute

KV-cache reuse

A shared context is encoded once and reused across turns instead of re-read every time: 20,263 → 4,099 prefill tokens over 5 turns on a 4,050-token context — with byte-identical output.

Measured on Apple M-series · Qwen2.5-3B-Instruct Q4_K_M · crucible-mini suite, seeds 0/1/2. Reproduce with crucible's benchmark harness — see Benchmarks & methodology.
