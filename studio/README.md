# Crucible Studio

A desktop control room for local LLM agents — the friendly face on top of
[Crucible Core](../). It detects your hardware, recommends and downloads a model
that fits, and runs the **OpenAI-compatible `crucible serve`** server with one
click, so opencode / Continue / Cursor / the `openai` SDK can point at it.

> Studio does **not** run inference itself. It reuses Product 1: the Python
> runtime (engine + grammar + server) is the brain; this Tauri app is the face.
> The native parts — hardware detection, model downloads, supervising the
> server — are Rust.

**v1 (this milestone): the model manager.** Hardware detection, model
recommendations, downloads with progress, and start/stop server control. A chat
panel that shows guaranteed-valid tool calls and live token savings is the next
milestone.

## Prerequisites

- **Rust** (stable) — <https://rustup.rs>
- **Node.js 18+** and npm
- **Crucible Core** (the `crucible` CLI on your PATH) — needed for **development**
  runs, where Studio launches it. **Released installers bundle the runtime** as a
  sidecar, so end users need nothing extra. See the
  [main install guide](../website/docs/user/install.md).
- Platform webview/build deps per the Tauri guide: <https://tauri.app/start/prerequisites/>
  (macOS just needs Xcode Command Line Tools).

## Run it (development)

```sh
cd studio
npm install
npm run tauri dev
```

The first run compiles the Rust backend (a few minutes); subsequent runs are fast.

## Build a desktop app

```sh
npm run tauri build
```

Output (`.app` / `.dmg` on macOS, etc.) lands in `src-tauri/target/release/bundle/`.

> **Icons:** placeholder icons are generated by `npm run gen-icons` (also runnable
> standalone). For a polished release, swap in a real logo with
> `npm run tauri icon path/to/logo.png`.

## One-download builds (bundled runtime)

So an end user needs **nothing else** — no Python, no Crucible Core — the released
app ships the runtime as a **Tauri sidecar**: a single frozen binary (Python +
the Crucible runtime + llama.cpp) built by [`scripts/build-sidecar.sh`](scripts/build-sidecar.sh).

At runtime, `server.rs` prefers that bundled binary (next to the app executable);
if it isn't there — e.g. a `tauri dev` build — it falls back to a `crucible` CLI on
your PATH. So development needs Crucible Core installed; a release does not.

To build a self-contained app locally:

```sh
bash scripts/build-sidecar.sh          # freezes the runtime into src-tauri/binaries/
npm run tauri build -- --config src-tauri/tauri.bundle.conf.json
```

In CI, [`studio-release.yml`](../.github/workflows/studio-release.yml) does this on
macOS and Linux automatically on a `v*` tag.

> **Status (honest):** the sidecar *wiring* is done and the dev path is verified.
> The freeze itself (PyInstaller bundling `llama-cpp-python` with its native backend
> / Metal shaders) is the finicky part and is validated by the first real build — it
> may need per-OS tuning. **Windows** doesn't freeze a sidecar yet, so the Windows
> installer falls back to a `crucible` CLI on PATH until that's added.

## How it fits together

```
┌──────────────────────── Crucible Studio (Tauri) ────────────────────────┐
│  React + Tailwind UI  ──invoke──▶  Rust core                            │
│   · system panel                   · sysinfo (RAM / CPU / GPU)          │
│   · model catalog                  · GGUF download (progress events)    │
│   · server control                 · spawn/stop `crucible serve`        │
└───────────────────────────────────────────┬─────────────────────────────┘
                                             │ spawns
                                             ▼
                         crucible serve  →  http://127.0.0.1:8000/v1
                         (Product 1: engine + grammar + OpenAI API)
```

## Project layout

```
studio/
├── src/                 # React + Tailwind frontend
│   ├── lib/api.ts       # typed wrappers over the Tauri command bridge
│   └── components/       # SystemPanel, ModelCatalog, ServerControl, ui/*
├── src-tauri/           # Rust backend
│   └── src/
│       ├── system.rs    # hardware introspection
│       ├── catalog.rs   # model catalog + recommendation
│       ├── download.rs  # streaming GGUF download + progress events
│       └── server.rs    # start/stop/inspect `crucible serve`
└── scripts/gen-icons.mjs
```
